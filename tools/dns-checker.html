What is DNS propagation?

The Domain Name System (DNS) is the internet's address book: it turns a human name like example.com into the IP address a machine actually connects to. When you change a DNS record, that update starts at your authoritative name servers and has to spread to the thousands of recursive resolvers around the world that answer queries for everyone else. Propagation is that spreading — and it isn't instant.

Each record carries a TTL (time to live) in seconds. A resolver is allowed to cache the old answer until the TTL expires, so a record with a 3600-second TTL can keep serving the previous value for up to an hour after you change it. This tool asks several independent public resolvers the same question at once and shows you who has the new answer and who is still serving the old one.

Common DNS record types

TypePurpose
AMaps a name to an IPv4 address.
AAAAMaps a name to an IPv6 address.
CNAMEAlias — points one name at another name.
MXMail exchange — where email for the domain is delivered.
NSThe authoritative name servers for the domain.
TXTArbitrary text — SPF, DKIM, domain verification, and more.
SOAStart of authority — the zone's primary server and serial number.
CAAWhich certificate authorities may issue certs for the domain.
SRVLocates a service (host and port) for a protocol.
PTRReverse lookup — maps an IP back to a name.

Why do resolvers disagree?

Right after a change, some resolvers have already fetched the new record while others are still inside the old TTL window — so they legitimately return different answers for a while. Geographic anycast, split-horizon DNS, and aggressive caching by ISPs can also cause differences. If every resolver above agrees, your change has fully propagated. If they don't, give it up to the record's TTL and check again.

Check DNS from the command line

{% raw %}# Query a record with dig (Linux / macOS)
dig example.com A +short
dig example.com MX +short
dig NS example.com +short

# Ask a specific resolver (here, Google & Cloudflare)
dig @8.8.8.8 example.com A +short
dig @1.1.1.1 example.com A +short

# See the TTL still counting down
dig example.com A            # look at the number before the record

# Windows
nslookup -type=A example.com
nslookup -type=MX example.com 8.8.8.8

# Reverse (PTR) lookup for an IP
dig -x 8.8.8.8 +short{% endraw %}
