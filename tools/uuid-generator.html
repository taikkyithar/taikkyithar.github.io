What is a version 4 UUID?

A UUID (Universally Unique Identifier) is a 128-bit value written as 32 hexadecimal digits in five hyphen-separated groups — 8-4-4-4-12. A version 4 UUID is the random one: 122 of its 128 bits come straight from a random source, with the remaining 6 bits reserved to mark the version and variant.

That's why every UUID on this page has a 4 at the start of the third group, and an 8, 9, a, or b at the start of the fourth:

3f2a7c4e-91b5-4d8a-a7e1-6c0b93d5f284
                    ^         ^
                 version    variant

Are they really unique?

Not guaranteed — but the odds are small enough to ignore. With 122 random bits there are about 5.3 × 1036 possible values. You'd need to generate a billion UUIDs per second for roughly 85 years to reach a 50% chance of a single collision.

The one thing that matters is the quality of the randomness. This tool uses your browser's crypto.randomUUID(), falling back to crypto.getRandomValues() — both cryptographically secure. It never uses Math.random(), which is not.

When to use v4 (and when not to)

  • Good for: primary keys generated client-side, correlation and trace IDs, idempotency keys, file and object names, anything that needs to be unique without coordinating with a central server.
  • Think twice for: clustered database primary keys. Random UUIDs scatter writes across the index, which hurts insert performance and page locality — UUIDv7 or a sequential ID is usually the better fit there.
  • Not for: anything secret. A UUID is an identifier, not a credential. Don't use one as a session token, password-reset token, or API key.
